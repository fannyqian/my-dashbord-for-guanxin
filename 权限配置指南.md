# 企业微信文档API权限配置指南

## 问题描述
运行 `fetch_docs.py` 时遇到错误：
```
{'errcode': 48002, 'errmsg': 'api forbidden'}
```

这表示当前应用没有调用企业微信文档API的权限。

## 解决步骤

### 1. 登录企业微信管理后台
访问：https://work.weixin.qq.com/

### 2. 找到对应的应用
- 进入"应用管理"
- 找到使用 Corp Secret `QEmipYLHRVvYHXOi2nVq6igk4oGBmVo7j_9lnA6fHXY` 的应用
- 或者根据 Corp ID `wwe032718ce9485b7d` 查找

### 3. 配置API权限
在应用详情页面，需要开通以下权限：

#### 必需权限
- **微文档** - 文档管理权限
  - `wedoc:doc_get` - 获取文档基本信息
  - `wedoc:doc_get_content` - 获取文档内容
  - `wedoc:smartsheet_get` - 获取表格内容

#### 可能的权限位置
- "接口权限" 或 "API权限"
- "微文档" 或 "企业微信文档"
- 可能需要管理员审批

### 4. 确认文档访问权限
确保应用有权限访问这些文档：
- 系统相关知识库 (AI4A4AcKABA)
- 门店个案知识库 (AI4A4AcKABA)
- 单独QA文档 (AXAAoQYrAIQCN61pqVd8uSQWwhAqa)

可能需要：
- 将应用添加到文档的可见范围
- 或者将文档设置为"企业内可见"

### 5. 测试权限
权限配置完成后，运行测试：
```bash
python fetch_docs.py
```

如果成功，应该看到：
```
[OK] Access Token获取成功
[OK] 已保存到: ./raw_docs/系统相关知识库.txt
[OK] 已保存到: ./raw_docs/门店个案知识库.txt
[OK] 已保存到: ./raw_docs/单独QA文档.txt
```

## 常见问题

### Q1: 找不到"微文档"权限选项
A: 可能需要先在企业微信中启用"微文档"功能，或者联系企业微信管理员开通。

### Q2: 权限已开通但仍然报错
A:
- 检查 Corp Secret 是否正确
- 确认应用是否有文档的访问权限
- 尝试重新生成 Access Token

### Q3: 不同文档类型需要不同权限吗？
A: 是的，wiki文档和表格文档可能需要不同的API权限，建议全部开通。
